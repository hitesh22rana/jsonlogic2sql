# WASM Playground

The library can be compiled to WebAssembly and run entirely in the browser. This enables an interactive playground where users can try JSONLogic-to-SQL transpilation without any server-side components.

## Quick Start

```bash
# Build the WASM binary
make build/wasm

# Serve the demo locally
cd demo/wasm && python3 -m http.server 8080
# Open http://localhost:8080
```

## How It Works

The Go library is compiled to WASM using Go's native `GOOS=js GOARCH=wasm` target. A thin Go wrapper (`demo/wasm/main.go`) bridges the library's public API to JavaScript via `syscall/js`, exposing functions on a global `jsonlogic2sql` object.

```
Browser
  └─ index.html loads wasm_exec.js (Go WASM runtime glue)
  └─ index.html fetches jsonlogic2sql.wasm (compiled from main.go)
       └─ main.go imports the jsonlogic2sql Go library
       └─ registers functions via syscall/js
  └─ JavaScript calls jsonlogic2sql.transpile(), etc.
```

## Files

| File | Generated | Description |
|------|-----------|-------------|
| `demo/wasm/main.go` | No | Go source - bridges library API to JavaScript |
| `demo/wasm/index.html` | No | Interactive demo UI |
| `demo/wasm/jsonlogic2sql.wasm` | Yes | Compiled WASM binary (~3.6 MB, ~1 MB gzipped) |
| `demo/wasm/wasm_exec.js` | Yes | Go's WASM runtime glue, copied from `$(go env GOROOT)/lib/wasm/` |

The `.wasm` and `wasm_exec.js` files are generated by `make build/wasm` and should not be edited manually. The `wasm_exec.js` file is version-coupled to the Go toolchain - rebuilding via `make build/wasm` keeps it in sync.

## JavaScript API

### Quick Usage (no instance management)

```javascript
const result = jsonlogic2sql.quickTranspile('bigquery', '{"==": [{"var": "status"}, "active"]}');
// result = { sql: "WHERE status = 'active'" }
```

### Full API

```javascript
// Create a transpiler instance for a specific dialect
const t = jsonlogic2sql.newTranspiler('bigquery');
// t = { id: 1 }

// Transpile JSONLogic to SQL (with WHERE prefix)
const result = jsonlogic2sql.transpile(t.id, '{"==": [{"var": "status"}, "active"]}');
// result = { sql: "WHERE status = 'active'" }

// Transpile condition only (without WHERE prefix)
const result = jsonlogic2sql.transpileCondition(t.id, '{"==": [{"var": "status"}, "active"]}');
// result = { sql: "status = 'active'" }

// Set schema for field validation and type coercion
const schemaResult = jsonlogic2sql.setSchema(t.id, JSON.stringify([
    { name: "status", type: "string" },
    { name: "amount", type: "integer" }
]));
// schemaResult = { ok: true }

// With schema, type coercion is applied automatically
const coerced = jsonlogic2sql.transpile(t.id, '{"in": [{"var": "status"}, [5960, 9000]]}');
// coerced = { sql: "WHERE status IN ('5960', '9000')" }

// Get list of supported dialects
const dialects = jsonlogic2sql.getDialects();
// dialects = ["bigquery", "spanner", "postgresql", "duckdb", "clickhouse"]

// Get sample JSONLogic expressions
const samples = JSON.parse(jsonlogic2sql.getSamples());
// samples = [{ name: "Simple equality", jsonLogic: '{"==": ...}' }, ...]

// Clean up
jsonlogic2sql.destroyTranspiler(t.id);
```

### Error Handling

On error, functions return an object with `error` (and optionally `code`, `operator`, `path`):

```javascript
const result = jsonlogic2sql.transpile(t.id, 'not valid json');
// result = { error: "[E007]: invalid JSON: ...", code: "E007", operator: "", path: "" }
```

## Supported Dialects

Pass these strings to `newTranspiler()` or `quickTranspile()`:

| String | Dialect |
|--------|---------|
| `bigquery` | Google BigQuery |
| `spanner` | Google Cloud Spanner |
| `postgresql` | PostgreSQL |
| `duckdb` | DuckDB |
| `clickhouse` | ClickHouse |

## Playground Features

The included `index.html` demo provides:

- **Dialect picker** - switch between all 5 SQL dialects
- **Compare all dialects** - view output for all dialects side-by-side
- **Schema support** - import from JSON file, load a sample schema, or type manually
- **WHERE vs Condition toggle** - switch between `WHERE status = 'active'` and `status = 'active'`
- **Sample expressions** - 10 built-in examples covering common patterns
- **Live transpilation** - output updates as you type
- **Error display** - structured error messages with error codes

## Build Details

- **Compilation**: `GOOS=js GOARCH=wasm go build`
- **Binary size**: ~3.6 MB uncompressed, ~1 MB gzipped
- **Dependencies**: Zero external dependencies (stdlib only)
- **WASM compatibility**: No CGo, no syscalls, no network/filesystem (except `NewSchemaFromFile` which is not exposed to WASM)

## Using in Your Own Page

To embed the transpiler in your own web page:

```html
<script src="wasm_exec.js"></script>
<script>
    const go = new Go();
    fetch('jsonlogic2sql.wasm')
        .then(r => r.arrayBuffer())
        .then(bytes => WebAssembly.instantiate(bytes, go.importObject))
        .then(result => {
            go.run(result.instance);
            // jsonlogic2sql is now available globally
            const sql = jsonlogic2sql.quickTranspile('bigquery', '{"==": [{"var": "x"}, 1]}');
            console.log(sql); // { sql: "WHERE x = 1" }
        });
</script>
```
